# AI Meeting Minutes & Compliance Report Generator

Turns a raw meeting transcript into a structured, compliance-style report — generated by an LLM and rendered in a polished, page-based report UI.

Built for the *AI Meeting Management Platform* take-home assignment. Scope: the transcript → AI → report-UI pipeline, not the full multi-tenant product described in the reference proposal (no auth, billing, or admin panel — see [Scope](#scope) below).

## What it does

1. Accepts a raw meeting transcript (paste or upload — plain text / `.docx`).
2. Sends it to an LLM with a prompt engineered to return **structured JSON**: meeting metadata, attendee list, executive summary, context, agenda/discussion log with speaker attribution, decisions & open items, and votes.
3. Renders that JSON into a designed report — cover page, document details, executive-summary cards, attendance table, role-coded speaker bubbles, color-coded alert callouts (decision / unresolved / tension / projection), timeline, and vote blocks.
4. Presents the finished report in a page-based viewer (sidebar table of contents + page navigation), plus a separate compliance dashboard (score, risks, findings table).

The design system started from the client-provided `Report_Design_Template_Blue.html`, prototyped with v0 + 21st.dev components, then wired to a real backend — see [Design decisions](#design-decisions).

---

## Scope

**In scope:** transcript input → AI report generation → report viewer → compliance dashboard.

**Out of scope** (belongs to the long-term product roadmap in the client's proposal, not this assignment): user registration/auth, tiered pricing/quotation flow, admin panel, Deepgram transcription integration, voice agent, CRM. The generated report also isn't a full reproduction of the 148-page reference PDF — it's a representative report that exercises every component type shown in the reference design template.

## How report generation works

1. **Input** — transcript submitted from `components/forms/NewReportForm.tsx` (paste or upload), along with meeting metadata.
2. **Generation** (`app/api/generate-report/route.ts`) — the transcript + metadata are sent to the LLM using a custom prompt, which asks for **JSON only** matching the schema in `lib/schema.ts`.
3. **Validation** — the response is parsed against the Zod schema before it reaches the client.
4. **Rendering** — the validated JSON is passed straight into the report and dashboard components. Report language mirrors the source transcript (French in the reference case).

## Design decisions

- Frontend built with Next.js App Router and Tailwind CSS, porting styling tokens (colors, font hierarchy) from `Report_Design_Template_Blue.html`.
- Light theme, single confident accent color, generous whitespace — aimed at a modern SaaS feel (Linear/Vercel/Notion-adjacent) rather than a dense enterprise-admin look.
- Kept the reference template's color-coded semantics (decision/unresolved/tension/projection, compliance status colors) since that mapping is what makes the report scannable at a glance.
- Generated a shorter, representative report that exercises every component type shown in the template's component library, demonstrating the pipeline capability clearly.
